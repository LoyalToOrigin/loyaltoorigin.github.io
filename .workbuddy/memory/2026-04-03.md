# 2026-04-03

## Hexo 博客部署修复

解决了两个问题成功部署博客：

1. **SSL 连接超时**: `_config.yml` 中的 `deploy.repo` 从 HTTPS (`https://github.com/...`) 改为 SSH (`git@github.com:...`)，因为用户网络环境 HTTPS 访问 GitHub 超时，但 SSH 正常。
2. **Node.js 版本不兼容**: 用户默认 Node.js v25.6.1，Hexo 3.9.0 的 `hexo-deployer-git` 依赖的 `graceful-fs` 报 `mode must be int32` 错误。通过 `nvm use 16` (v16.20.2) 并升级 `hexo-deployer-git@latest` 解决。
3. 同时修正了 `_config.yml` 中 `url` 字段（原来是 git 仓库地址，改为网站访问地址 `https://loyaltoorigin.github.io`）。

### 部署命令
```bash
$HOME/.nvm/versions/node/v16.20.2/bin/npx hexo clean
$HOME/.nvm/versions/node/v16.20.2/bin/npx hexo g
$HOME/.nvm/versions/node/v16.20.2/bin/npx hexo d
```

### 注意事项
- 有几篇文章 front matter 解析有 ERROR（isDate is not a function / YAML 格式问题），但不影响部署，这些文章未被生成

## Hexo 升级到 8.1.1

成功将 Hexo 从 3.9.0 升级到 8.1.1，修复了所有问题：

### 升级内容
- Hexo 核心：3.9.0 → 8.1.1
- 所有插件升级到最新兼容版本
- Node.js 锁定到 v16.20.2（通过 `.nvmrc` 文件）

### 网络问题修复（关键）
**问题**：DNS 污染导致 `github.com` 解析到错误 IP `198.18.0.25`（RFC 2544 测试网段），SSH/HTTPS 连接均失败。

**解决**：在 `~/.ssh/config` 中直接指定 GitHub 真实 IP（20.205.243.166）：
```ssh
Host github.com
    HostName 20.205.243.166
    User git
    Port 22
    StrictHostKeyChecking no
    UserKnownHostsFile /dev/null
```

### 其他修复
- 创建 `.deploy_gitignore` 文件，防止 `.deploy_git` 目录被错误包含进部署
- 修复了 GRDB 文章的 front matter（缺少 YAML 头部）
- 移除了 `_config.yml` 中的 `extend_dirs: /` 配置，避免将整个项目源码部署到 GitHub Pages

### 当前部署命令
```bash
export NVM_DIR="$HOME/.nvm"
source "$NVM_DIR/nvm.sh"
nvm use 16
npx hexo clean && npx hexo g && npx hexo d
```
